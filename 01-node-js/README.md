# 01 - Node.js

Node.js is a JavaScript runtime that allows us to run JavaScript outside the browser.

Normally JavaScript runs in the browser, but with Node.js we can use JavaScript to build backend servers, APIs, scripts, command-line tools, and real-time applications.

Node.js is built on the V8 JavaScript engine, which is also used by Google Chrome.

## Simple Definition

Node.js is a runtime environment that lets us run JavaScript on the server side.

## Why We Use Node.js

- To build backend APIs using JavaScript
- To run JavaScript outside the browser
- To handle many requests efficiently using non-blocking I/O
- To build tools, scripts, and server-side applications
- To use the same language on frontend and backend

## Summary

Node.js is a JavaScript runtime built on the V8 engine. It allows us to run JavaScript outside the browser, mostly for backend APIs, servers, scripts, and real-time applications.

## Practice Questions

### Q1. What is Node.js?

Node.js is a JavaScript runtime built on Chrome's V8 engine that allows us to run JavaScript outside the browser.

### Q2. Why do frontend engineers need to learn Node.js?

Frontend engineers learn Node.js because it allows us to use JavaScript on the server side, so we can build APIs, backend logic, scripts, and become full-stack developers more easily.

### Q3. What is the main difference between JavaScript in the browser and JavaScript in Node.js?

Browser JavaScript runs inside the browser and can access browser APIs like DOM, window, and document. Node.js JavaScript runs outside the browser and can access server-side features like file system, operating system, and backend APIs.
